# 开发记录 #07 — 渐进绘制动画

- **日期**：2026-06-12
- **分支**：`feat/07-progressive-draw`
- **对应路线图**：PR #7（试用反馈新增需求："能不能像画画一样一点一点画出来"）

## 做了什么

`src/canvas/draw-animation.ts`：新图形不再瞬间出现，而是分两阶段"手绘"出来：

1. **描边阶段（0.5s）**：虚线偏移技巧——把整条轮廓设为一段与周长等长的虚线，
   dashOffset 从周长动画到 0，视觉上即"一笔画出轮廓"。周长按图形类型精确计算
   （圆 2πr / 矩形 2(w+h) / 正多边形 n·2r·sin(π/n) / 折线逐段累加）。
2. **填充阶段（0.25s）**：填充色从全透明 rgba 淡入到目标色。直线无填充，只走描边阶段。

执行层配合改动：填充图形创建时带同色描边（视觉无差异），动画沿描边轨迹进行。

## 关键设计约束

**动画是纯视觉表现，不进入逻辑层**：节点几何在创建瞬间即为最终状态，
对象登记、选中命中、移动、删除在动画进行中全部正确工作；
执行反馈（字幕气泡）也即时给出。因此"语音→首笔反馈"延迟指标不受影响
（评分维度二），demo 观感却接近真人作画。

## 验证方式

```
node scripts/e2e-anim.mjs   # 前置 npm run dev
```

实测采样（画一个红色的大圆，周长 515px）：

| 时刻 | dashOffset | 填充 |
|---|---|---|
| ~60ms | 503 | 透明 |
| ~200ms | 312 | 透明 |
| ~400ms | 0（描边完成） | alpha 0.07（淡入中） |
| ~1000ms | — | alpha 1（完成） |
