# 开发记录 #12 — 容错强化：相对定位、缺图形追问、口语动词

- **日期**：2026-06-12
- **分支**：`feat/12-robust-parse`
- **背景**：对照外部功能测试规格（P0/P1/P2/SYS），把**不依赖 AI** 就能实现的
  P1 项全部落地（AI 依赖边界已写入 design.md §4.4）。

## 做了什么

1. **相对定位 relativeTo**（P1 能力 #13，TC-P1-02）：
   - DSL：`props.relativeTo = {shape?, color?, relation}`（relation: left-of/right-of/above/below），
     schema 校验（锚点至少一个特征）；
   - 解析："在圆的右边画一个方形"——锚点段先行剥离再提取新图形槽位
     （"在红色的圆左边画个蓝色三角"中锚点的红色不会泄漏给新图形）；
   - 执行：按特征找锚点（多匹配取最近创建），包围盒 + 间距 + 新对象大小
     算出中心，转回比例坐标，越界钳到画布内；锚点缺失明确报错
     "画布上没找到要参照的三角形"，不静默乱放（design.md §3.3 原则）；
   - LLM 提示词同步（LLM 也可以产出 relativeTo，比硬猜 fx/fy 更稳）。
2. **规则级缺图形追问**（P1 能力 #14，TC-P1-05 / TC-SYS-05）：
   - 判定 `isShapeMissing`：有绘制动词、无图形词、去掉指代/量词后**零剩余信息**
     （"画一个"、"画那个那个那个"）→ 追问"想画什么图形呢？"——
     注意"画一个人"剩余"人"，仍交给 LLM，不误拦；
   - 回答补上"画"字重新解析（"红色的三角形"→"画红色的三角形"，
     颜色/大小/位置槽位都生效）；取消词放弃；无关输入按新指令处理；
   - 走追问面板 + TTS 播报问题，无 Key 也可用，命中时省一次 LLM 调用。
3. **口语动词与同义词**：绘制动词补"搞/弄"（TC-P1-04），图形词补"球"→circle。
