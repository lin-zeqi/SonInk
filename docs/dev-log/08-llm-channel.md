# 开发记录 #08 — LLM 复合指令拆解通道（慢路径）+ 追问面板 + 设置面板

- **日期**：2026-06-12
- **分支**：`feat/08-llm-channel`
- **对应路线图**：原 PR #10，按试用反馈提前（"画一个人现在做不到，复杂的要对接 AI 并能追问"）

## 做了什么

1. **DSL 扩展（先行）**：组合图形需要比九宫格更细的摆放粒度——
   - `position` 新增 0~1 画布比例坐标 `{"fx":0.5,"fy":0.28}`；
   - 直线新增 `from`/`to` 起止点（此前只能画水平线，火柴人的四肢画不出来）；
   - **像素绝对坐标仍然禁止**（设计原则不变，比例坐标可校验、无幻觉压力），
     schema 同步校验（越界即拒）。design.md §3.3 已记录这次修订及理由。
2. **LLM 客户端**（src/llm/client.ts）：DeepSeek chat API，JSON 模式，temperature 0.2。
3. **系统提示词**（src/llm/prompt.ts）：教 LLM 输出本项目 DSL；信息不足时返回
   `{"ask":"问题"}` 而非乱画；内置火柴人拆解示例；追问最多两轮。
   **DSL 变更时必须同步更新此文件**。
4. **会话状态**（src/store/assistant.ts）：thinking / ask / messages 上下文。
   追问后用户的下一条指令作为回答续入同一轮会话（带完整上下文）。
5. **管道路由**（pipeline.ts，改为异步）：
   - 规则命中 → 本地执行（快路径不变，延迟不受影响）；
   - 规则未命中 → LLM；LLM 输出过 validateDsl 校验后才执行，校验失败不执行；
   - 追问等待中 → 输入作为回答续话；"取消/算了"退出会话；
   - 未配 Key → 反馈引导去设置面板。
6. **追问面板**（AskPanel.vue）：画布上方居中弹出，醒目展示 AI 的问题
   （试用反馈明确要求：追问要用提示弹出，不能只在底部气泡里）；
   另有"AI 思考中…"状态提示。
7. **设置面板**（SettingsPanel.vue）：顶栏"设置"按钮展开，填 DeepSeek API Key，
   只存 localStorage，永不入仓库。

## 验证方式

```
node scripts/e2e-llm.mjs   # 前置 npm run dev，不需要真实 Key
```

脚本拦截 DeepSeek API 请求返回模拟响应，验证完整闭环（实测通过）：

1. 输入"画一个人" → 规则未命中 → LLM（消息数 2：system+user）→ 返回 ask
   → **追问面板弹出**"想画多大的小人？…"；
2. 输入"中等大小，画中间就行" → 续话（消息数 4，含上下文）→ 返回 commands
   → 画出 1 圆 + 4 线的火柴人（截图确认成型）；
3. LLM 共调用 2 次，符合预期。

`npm run build` 通过；26 个解析冒烟用例不受影响。

## 追加：多服务商可选（同分支第二次提交）

设置面板从"单一 DeepSeek Key"升级为**服务商可选**：

- 预置 DeepSeek / Kimi（月之暗面）/ 智谱 GLM / 通义千问，外加"自定义"
  （任意 OpenAI 兼容端点 + 模型名）——国产大模型基本都兼容 OpenAI
  chat/completions 协议，客户端只需要 baseUrl/model/key 三元组；
- 各服务商 Key 分开保存、切换不丢；模型名可覆盖预设默认值；
- 旧版单一 DeepSeek Key 自动迁移到新配置结构；
- e2e 增加场景二：配置切到 Kimi 后请求正确路由到 api.moonshot.cn（mock 验证通过）。
