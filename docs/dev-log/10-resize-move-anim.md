# 开发记录 #10 — 缩放指令 + 移动/缩放过渡动画

- **日期**：2026-06-12
- **分支**：`feat/10-resize-move-anim`
- **背景**：试用反馈两点——"指令移动是瞬移的，没有拖拽效果"、"不能放大缩小"。
  （反馈中"不接 API 连矩形都画不出"经排查为远端 main 尚未合并 PR #5+，
  规则引擎本就支持"画一个矩形"，e2e 已复核。）

## 做了什么

1. **DSL 新增 `resize` 指令**（types/schema/executor/LLM 提示词四处同步）：
   `{action:'resize', target?, scale?, size?}`——scale 为倍数（>1 放大），
   size 为绝对目标半径；至少其一，schema 校验。
2. **规则解析**：放大/缩小/变大/变小/调大/调小/大一点/小一些… → resize；
   "放大一倍"=2、"缩小一半"=0.5、"一点/一些"=±25%、默认 1.5/0.67、
   "放大到半径五十"走绝对大小。意图判定排在绘制之后（"画一个大一点的圆"仍是绘制）、
   移动之前（"放大"不会误中"放到"）。
3. **缩放执行**：圆/三角按半径，矩形缩宽高并补偿左上角（保持中心不动），
   直线用 scale 变换（首次缩放时把变换原点移到线段中心，位置补偿视觉不变）。
   下限半径 4px 防缩没。多匹配时拒绝并提示先选中（与移动一致）。
4. **过渡动画**：移动/缩放统一 `animateTo`——Konva Tween 0.35s EaseInOut，
   动画期间清除高亮框、结束后按选中态重绘。纯视觉：几何终态在指令执行时即确定。
5. **待定终态机制**（撤销与动画的配合，关键修正）：动画异步进行，
   `executeAll` 紧随其后的快照比较会读到中间值——移动/缩放会被误判
   "状态没变"而不进撤销历史。`snapshot.ts` 增加 pendingAttrs 登记：
   执行引擎启动动画时登记终态属性，快照捕获时用终态覆盖中间值，
   动画结束清除；restoreSnapshot 时整体作废（旧节点连动画一起销毁）。
