# 开发记录 #09 — 撤销 / 重做（事务快照历史栈）

- **日期**：2026-06-12
- **分支**：`feat/09-undo-redo`
- **对应路线图**：PR #9

## 做了什么

1. **事务快照替代命令模式**（设计修订，design.md §3.4 已记录）：
   原计划每条指令封装 `{ apply(), revert() }`。实现时改为整体快照——
   `executeAll` 执行变更类指令前用 Konva 的 `toObject()` 序列化画布
   （节点 + 对象登记表 + 选中态），撤销即 `Konva.Node.create` 整体恢复。
   理由：删除/清空的逆操作本就要保存完整节点，快照统一了所有指令的回滚路径，
   且复合指令天然作为一个事务。
2. **历史栈**（src/store/history.ts + src/history/snapshot.ts）：
   - undoStack 存"执行前"快照，深度上限 50；新变更提交后清空 redoStack（标准编辑器语义）；
   - 仅 draw/move/delete/clear 入历史（select 只改高亮不入栈，状态无变化的执行也不入栈）；
   - 快照捕获时清除渐进动画中间态（dash 残留、半透明填充按登记色还原为最终态）。
3. **复合指令整体语义**：
   - 一次"撤销"整体回退复合指令的全部子指令；
   - **失败整体回滚**：复合指令中途失败时恢复执行前快照，画布不停在半完成状态，
     反馈追加"（复合指令已整体回滚）"。
4. **DSL 新增 `undo`/`redo` 指令**（types/schema/executor/LLM 提示词四处同步），
   规则解析新增同义词：撤销/撤回/回退/上一步/悔棋 → undo；重做/取消撤销/恢复 → redo
   （重做先于撤销判定，否则"取消撤销"误中 undo）。
5. **三个入口同一管道**：语音（或调试输入）"撤销"、顶栏 ↩/↪ 按钮（按栈空置灰）、
   Ctrl+Z / Ctrl+Y / Ctrl+Shift+Z 快捷键——按钮与快捷键提交"撤销/重做"文本走指令管道，
   反馈行为与语音完全一致。输入框聚焦时快捷键不劫持（保留文本编辑的原生撤销）。
6. 顺带重构：`syncHighlight`/`findNode` 从 executor 抽到 `src/canvas/highlight.ts`
   （快照恢复也要重绘高亮，避免循环依赖）。
