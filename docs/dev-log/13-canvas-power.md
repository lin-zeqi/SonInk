# 开发记录 #13 — 语义模板、文本标注、改色、导出、回放与拖拽

- **日期**：2026-06-12
- **分支**：`feat/13-canvas-power`
- **背景**：测试规格 P2 中全部**不依赖 AI** 的能力一次落地（边界见 design.md §4.4）。

## 做了什么

1. **语义对象模板**（TC-P2-01 / TC-P1-01）：太阳（圆+8 光线）/房子/树/雪人/
   小人（火柴人）/笑脸，规则层直接展开为基础绘制指令序列——零延迟、无 Key 可演示。
   支持位置与大小修饰（"在左上角画一棵树"、"画一个大大的笑脸"）；模板词先剥离
   再提取槽位（"小人"的"小"不会误判为大小）；连接词拆分+动词承接兼容
   （"画一个房子，左边加一棵树，右上角有太阳"→ 13 个图形本地画出）。
   **副作用**：原走 LLM 的"画一个雪人"现被本地接管，e2e-llm 场景二改用"画一只小猫"。
2. **文本标注**（TC-P2-02）：DSL 新增 shape=text + props.text（schema 必填校验），
   "写上你好/在中间写上完成"；Konva.Text 居中定位、透明度淡入（无轮廓可描边）、
   默认黑色；"删掉文字"可按特征删除。
3. **改色**（TC-P2-03）：DSL 新增 style 指令；"把这个圆变成蓝色"——新颜色取改色
   动词之后、目标特征取动词之前（"把红色的圆涂成绿色"两个颜色不混淆）；
   颜色过渡动画；**登记表颜色同步**（改完可以"删掉蓝色的圆"）；可撤销。
4. **导出 PNG**（TC-P2-05）："保存图片"→ stage.toDataURL 2x 触发下载；
   导出前隐藏反馈层（选中高亮不进图）；DEV 模式暴露 lastExport 供 e2e 断言。
5. **历史回放**（TC-P2-06）："回放操作"→ undoStack 本身就是按时间排列的状态序列，
   逐个恢复（起点通常是空画布）最后回到当前状态，步进 700ms；
   回放中管道拒绝新指令；结束后画布与历史栈与回放前完全一致。
6. **鼠标拖拽**（TC-P2-04 前置）：图形创建即 draggable（属性随快照序列化，
   撤销恢复后仍可拖）；dragstart/dragend 挂在主图层（Konva 事件冒泡，
   快照重建的节点无需重新绑定），拖拽作为一次事务计入撤销历史。
