# 开发记录 #11 — TTS 播报 + 清空二次确认 + 操作历史面板

- **日期**：2026-06-12
- **分支**：`feat/11-tts-history`
- **对应路线图**：PR #11（P0 能力清单 #10、#11 收尾）

## 做了什么

1. **TTS 播报**（src/speech/tts.ts）：Web Speech SpeechSynthesis 封装，zh-CN，
   新播报打断旧播报。管道执行完成后播报结果；追问/确认场景播报**问题本身**
   而不是"见提示框"（纯语音交互闭环）。顶栏"🔊 播报开/关"可静音。
   - **自听回环防护**：播报开始时暂停语音识别、结束后恢复（onSpeakStateChange
     回调由 App 接线到 recognizer），避免系统把自己的播报当成用户指令。
2. **清空画布二次确认**（破坏性操作，design.md 反馈层规划项）：
   - 画布非空时"清空画布"先弹确认面板（橙色"需要确认"，复用 AskPanel）并播报问题；
   - 说/输入"确认"执行、"取消/算了"放弃、**无关输入按新指令处理**（确认自动解除）；
   - 面板上的确认/取消按钮提交文本走指令管道，与语音同路径；
   - 复合指令含清空（"清空画布然后画方块"）整个序列暂存、确认后整体执行；
   - 空画布清空免确认（无破坏即不打扰）。
3. **操作历史面板**（HistoryPanel.vue）：顶栏"历史"开关，倒序列出每条指令的
   文本、来源（语音/调试/按钮）与执行结果——执行结果由管道回填到指令流中枢
   （CommandEntry 新增 result 字段）。
4. 字幕条来源标签支持新增的"按钮"来源（撤销/重做/确认按钮提交的指令）。
