## 做了什么

1. **`src/parser/lexicon.ts`** — 词法资源：
   - 同义词表：图形（圆/圈/圈圈→circle 等 16 条）、颜色（13 条，映射 Material 色板）、
     大小（16 条语义词）、位置（24 条九宫格词）；
   - 所有表**长词在前**，首个命中生效，避免"圆形"被"圆"截胡、"左上角"被"左"截胡；
   - 归一化：语气词/礼貌前缀清洗（请/帮我/嗯/吧…）+ 标点清除；
   - 中文数字解析 `parseNumber`：支持到百位（"五十"→50、"一百二十"→120），
     兼容阿拉伯数字与混合写法。
2. **`src/parser/rules.ts`** — 规则引擎：
   - 意图识别：清空（清空/清除/重新开始…）与绘制（画/绘/来/加/添/整）；
   - 槽位提取顺序有讲究：**绝对大小先于语义大小**（"大小50"的"大"会被语义表误判，
     先匹配"半径/大小/尺寸+数字"模式并从文本中移除该片段再做后续匹配）；
   - 未命中返回 miss，由管道转交慢路径（LLM，后续 PR）。
3. **`src/pipeline.ts`** — 接入双通道路由：规则命中 → 执行；未命中 → 暂提示
   "没听懂"（LLM 通道占位）；DSL JSON 直通通道保留为调试手段。
4. **`scripts/parser-smoke.ts`** — 解析器冒烟测试脚本（13 个用例，node 直跑不依赖浏览器）。

## 设计决策记录

- **PR #4/#5 合并**：属性槽位与图形识别共用同一次文本扫描，拆成两个 PR 需要先交付
  一个"只认图形不认属性"的半成品解析器，无独立验证价值。
- **先抽绝对大小再抽语义大小**：消除"大小50"中"大"字的歧义，是本模块第一个
  真实容错案例。
- **MISS 而非猜测**：有绘制动词但无图形词（"画一个东西"）返回 miss 而不是默认圆形，
  宁可转交 LLM/提示没听懂，不静默画错。
